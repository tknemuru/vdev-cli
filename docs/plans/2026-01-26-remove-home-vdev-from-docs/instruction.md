[Goal]
vdev-cli の docs / ガイド文書から、廃止済みの ~/.vdev 運用（symlink 正本前提）の記述を削除し、現行の運用前提と矛盾しない説明に修正する。

[Risk Classification]
R3（高リスク）
理由：
- 本変更は「vdev の正しい使い方・正本の所在」に関するドキュメント修正であり、
  誤りがあると利用者の運用全体を恒常的に誤誘導する可能性があるため。
- 実装（CLI）変更は伴わないが、設計思想・SoT の理解に直接影響するため
  最終的な承認・マージは Human が責務を持つ。

[Background]
現在の docs に ~/.vdev/CLAUDE.md や ~/.vdev/vdev-flow.md、symlink 作成手順、グローバル正本として ~/.vdev を参照する説明が残っている。これは廃止済みの運用であり、読者を誤誘導するため削除したい。

[Scope]
1) リポジトリ内のドキュメント（例: docs 配下、ops ガイド、cheatsheet 等）に含まれる ~/.vdev 参照の削除
2) symlink 作成手順や「グローバル正本が ~/.vdev にある」前提の説明の削除
3) 必要な場合のみ、現行運用に沿う最小限の説明へ置換
4) 文書間の整合性を崩さない範囲で関連箇所を更新

[Non-Goals]
1) vdev CLI 実装仕様の変更
2) 新しい機能や追加の運用手順の提案
3) docs の大規模再構成（章立て変更、内容追加の拡張）

[Constraints]
1) ~/.vdev に依存する運用を前提とした説明は残さない
2) 置換文は断定しすぎず、現在の SoT と矛盾しない最小限に留める
3) 推測で新しいパスやセットアップ手順を作らない
4) 既存の用語や他文書参照（SoT への参照など）を壊さない

[Work Items]
1) リポジトリ全体で以下を検索し、該当箇所を洗い出す
   1.1) "~/.vdev"
   1.2) "vdev-flow.md" と "~/.vdev" の併記
   1.3) "CLAUDE.md" と "~/.vdev" の併記
   1.4) "symlink" と配布手順
   1.5) "global" "正本" "single source" 等の文脈で ~/.vdev を指す表現
2) 各文書で、以下の方針で修正する
   2.1) ~/.vdev のパス記載は削除
   2.2) symlink 作成手順ブロックは削除
   2.3) 「~/.vdev を正本として sync する」等の説明は削除または現行前提に沿う表現へ変更
   2.4) 章の意味が崩れる場合は、節ごと削除し、周辺の文章を自然につなぐ（新しい手順は増やさない）
3) 変更後、文書内で参照しているファイル名やリンク関係が破綻していないかを確認する
4) 変更点が分かるように、PR/コミットメッセージは「docs: remove ~/.vdev references」等の意図が明確なものにする

[Acceptance Criteria]
1) リポジトリ内のドキュメントから "~/.vdev" の記述が原則として消えている
2) 「symlink を作成する」手順が docs から消えている
3) 読者が ~/.vdev 前提のセットアップを行うよう誤誘導されない
4) 文書間参照（例: ops から SoT 参照など）が破綻していない

[Verify]
1) grep で残存を確認する
   1.1) grep -R "~/.vdev" -n .
   1.2) grep -R "ln -s" -n docs .
   1.3) grep -R "symlink" -n docs .
2) 修正した主要ドキュメントを目視で通読し、前後の文脈が自然であることを確認する

[Notes]
本トピックは docs の整合性修正のみを対象とし、vdev CLI の挙動や仕様（実装）を変更しない。
R3 指定のため、impl-review が DONE となった後の PR マージは Human が最終責務を持つ。
